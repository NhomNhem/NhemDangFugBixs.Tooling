# NhemDangFugBixs.Tooling - Project Context

## Project Overview

**NhemDangFugBixs.Tooling** is a Unity package (UPM) that provides a **VContainer Auto-Registration Source Generator** and **Roslyn Analyzers**. It automates dependency injection registration in Unity projects using simple attributes, eliminating boilerplate code.

### Core Components

| Component | Description |
|-----------|-------------|
| **Source Generator** (`DangFugBixs.Generators`) | Roslyn incremental generator that scans for `[AutoRegister]` and `[AutoRegisterFactory]` attributes, generating VContainer registration code at compile time. |
| **Runtime Library** (`DangFugBixs.Runtime`) | Contains attribute definitions (`AutoRegisterAttribute`, `AutoInjectSceneAttribute`) and supporting models for DI registration. |
| **Roslyn Analyzers** (`DangFugBixs.Analyzers`) | Real-time validation rules for VContainer best practices during development. |
| **Common Library** (`DangFugBixs.Common`) | Shared models (e.g., `ServiceInfo`, `ClassAnalyzer`) used across generator and analyzers. |

### Architecture

```
Source~/                                    # All source code (hidden from Unity)
├── DangFugBixs.Generators~/                # Source Generator (IIncrementalGenerator)
├── DangFugBixs.Runtime~/                   # Runtime attributes & logic
├── DangFugBixs.Analyzers~/                 # Roslyn Analyzers
├── DangFugBixs.Common~/                    # Shared models
└── NhemDangFugBixs.Tooling.sln             # Solution file

Runtime/                                    # UPM Runtime (DLLs only)
└── NhemDangFugBixs.Runtime.dll

Analyzers/                                  # UPM Analyzers (DLLs only)
├── NhemDangFugBixs.Analyzers.dll
└── NhemDangFugBixs.Generators.dll

Editor/                                     # Unity Editor utilities
└── SceneInjectedProcessor.cs               # Auto-syncs scene-injected components
```

## Building and Running

### Prerequisites

- **.NET 10.0+** (for tooling components)
- **Unity 2021.3+** (supports Roslyn Analyzers)
- **VContainer** installed in target Unity project

### Build Commands

```bash
# Restore dependencies
dotnet restore

# Build solution
dotnet build --no-restore

# Run unit tests
dotnet test --no-build --logger "console;verbosity=normal"

# Clean build
dotnet clean && dotnet build
```

### Deployment to Unity

The project uses an **MSBuild post-build target** to automatically deploy DLLs:

1. **Automatic Discovery**: Place a `.nhem-deploy-target` file in your Unity project root.
2. **Manual Override**: Set `NHEM_UNITY_PROJECT_ROOT` environment variable or create `LocalBuild.props`.
3. **Build Trigger**: Running `dotnet build` in `Source~` copies Runtime DLLs to `Assets/Plugins/`.

> **Note**: Analyzer/Generator DLLs are **not** copied to `Assets/`. Unity loads them directly from the package's `Analyzers/` folder to prevent duplicate execution.

## Development Conventions

### File Structure Rules

- **All `.cs` source files** must reside in `Source~/` to prevent Unity domain reload issues.
- **Only compiled DLLs** are placed in `Runtime/` and `Analyzers/`.
- **`.meta` files** are critical for UPM distribution—templates in `Source~/MetaTemplates/` are the source of truth.

### Namespace Convention

All code uses the `NhemDangFugBixs.*` namespace hierarchy:

```csharp
namespace NhemDangFugBixs.Attributes { }
namespace NhemDangFugBixs.Generators { }
namespace NhemDangFugBixs.Editor { }
```

### Code Style

- **Braces**: Egyptian style (opening brace on same line).
- **Braces mandatory** for all control flow blocks, even single-line.
- **Private fields**: Underscore prefix (`_camelCase`).
- **Nullable**: Enabled (`<Nullable>enable</Nullable>`).
- **Target Framework**: `net10.0` or higher for tooling components.

### Source Generator Best Practices

- **Incremental Generators**: Always inherit from `IIncrementalGenerator`.
- **Predicates**: Keep `CreateSyntaxProvider` predicates lightweight (syntax-only).
- **Heavy analysis** belongs in the `transform` phase.
- **Code emission**: Use `IndentedTextWriter` with `Block` extension for consistency.
- **Generated files**: Must include `// <auto-generated />` header.

### VContainer Integration

- **Attributes**:
  - `[AutoRegister(Lifetime)]` - Auto-registers class with VContainer.
  - `[AutoRegisterFactory]` - Registers `Func<T>` factory.
  - `[AutoInjectScene]` - Marks component for scene-based injection.
- **Registration Pattern**: `VContainerRegistration.RegisterGlobal(builder)` or `Register{ScopeName}`.

## Common Pitfalls

1. **TypeLoadException**: Never place `.cs` files in `Runtime/` or `Editor/`—use `Source~/`.
2. **CS0234 (Missing References)**: Ensure `DangFugBixs.Common` is referenced by both Generators and Runtime.
3. **Meta File Overwrites**: Unity may regenerate `.meta` files—always use templates from `Source~/MetaTemplates/`.
4. **Duplicate Generator Execution**: Do not copy Generator DLLs to `Assets/Plugins/`—Unity loads them from `Analyzers/` directly.
