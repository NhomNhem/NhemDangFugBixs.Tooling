# Contributing & Development (v6.0)

## Commit Rules
- Use: `feat: ...`, `fix: ...`, `chore: ...`
- Keep commit messages concise and scoped

## Adding Analyzer Rules
- Add analyzer to `DangFugBixs.Analyzers/`
- Add tests in `DangFugBixs.Analyzers.Tests/`
- Follow NDxxx code conventions

## Before PR
- Run `dotnet test` for all projects
- Ensure no new warnings or errors

## Support
- Open a GitHub issue for bugs, questions, or feature requests
